牛肉麵問題:¶

這碗牛肉麵好吃嗎？

瓦哥看著眼前的牛肉麵心想: 這碗牛肉麵好吃嗎？ 瓦哥近來致力烹煮牛肉麵，想要重製出印象中的美味。他詳細紀錄了自己最近烹煮的100次牛肉麵所使用的烹煮時間與爐火大小。資料長這樣。

In [1]:
import matplotlib.pyplot as plt
import numpy as np
% matplotlib inline

X = np.random.rand(100, 2)
cook_time =  np.hstack([0.3 + X[:20,0] + 0.2 * X[:20,1], 
                        X[20:,0] + 0.5 * X[20:,1]])

heat =  np.hstack([0.1 + X[:20,0] + X[:20,1], 
                            X[20:,0] + 0.2 * X[20:,1]])
y = np.array([1] * 20 + [0] * 80)
                           
                           
for i in range(2):
    plt.plot(cook_time[y==i], heat[y==i], '.')
plt.xlabel('cook time')
plt.ylabel('heat')
plt.legend(['not good', 'good'])
plt.show()

當然，瓦哥也標記了每一次練習成品的美味與否。20個綠色點是要命的好吃，剩下80個藍色則是普普惹。讓瓦哥困擾的是，有沒有辦法預測牛肉麵好不好吃呢？

瓦哥心想:林北每次也都亂煮，好不好吃都馬隨機，那就把這問題當成是丟銅板，好吃機率是p，不好吃機率就馬的1-p。更我真天才。

於是瓦哥寫下了他收集到資料的likelihood $$P =\Pi_{i=1}^{100} p^{y_i}(1-p)^{1-y_i} =p ^{20} (1-p)^{80}$$ $$log(P) = 20 \times log(p) + 80 \times log(1-p)$$

其中$y_i$ 是每次烹煮好不好吃結果。$y_i=1$表示好吃，$y_i=0$則是不好吃

身為瓦哥會解MLE也是剛好，瓦哥發現當$p = 0.2$ 時likehood會最大。

於是根據著個模型，瓦哥結論，下次烹煮牛肉麵會好吃的機率是0.2, 不好吃的機率是0.8。

想一想：¶

這模型是不是很廢？

根據這模型牛肉麵好吃的機率跟如何烹煮(烹煮時間與爐火大小)毫無關聯。有沒有辦法讓這個模型與烹煮方式相關連呢?

瓦哥怒而諮詢和他一起看食戟之靈的好友彭九。彭九建議他把p替換成 $$p(cooktime, cooktime) = 1/(1 + exp(a \times cooktime + b \times heat + c))$$ 其中$1/(1+exp(x))$ 就是傳說中的logistic function! 偉哉彭九！

我們來看一下logistic function長怎樣：

In [2]:
x = np.linspace(-10.0, 10.0, num=1000)
plt.plot(x, 1.0/(1.0 + np.exp(-x)))
plt.show()

瓦哥觀察到logistic function有幾個特點：

  1. 值都在0-1之間，符合對於機率的定義
  2. 當 x = 0, logistic function 的值 = 0.5
  3. 當x越大, logistic function的值也越大。

假如這個值代表的是好吃牛肉麵的機率，那麼我們也可以說，烹煮牛肉麵秘訣就是讓x越大！

回到彭九的建議: $$p(cooktime, cooktime) = 1/(1 + exp(a_1 \times cooktime + a_2 \times heat + b))$$ 彭九根據對於資料的觀察，認為烹煮時間和火的強度有某種線性因子 $F = a_1 \times cooktime + a_2 \times heat + b$, 當F值越大，烹煮出來的牛肉麵就越可能是好吃的。而在$F=0$時，好不好吃的機率則是各半,微妙微妙。再來看一下瓦哥的資料

紅線標出了 heat - cooktime = 0的線段。這樣大略的觀察似乎告訴我們可以假設 F = heat - cooktime,

  1. 當F>0(線段上方), 牛肉麵都是好吃的,
  2. 當F<0(線段下方)的牛肉麵則可能不好吃。

問題來了。上面我們純粹猜測F = heat - cooktime。有沒有更有系統的方法來尋找出這個線性因子呢?

瓦哥於是將彭九的insight $$p(cooktime, heat) = 1/(1 + exp(a_1 \times cooktime + a_2 \times heat + b))$$ 代入自己原先廢模型的likelihood得到一個新的objective function $$P(a_1,a_2,b) =\Pi_{i=1}^{100} p(cooktime_i, heat_i)^{y_i}(1-p(cooktime_i, heat_i))^{1-y_i}$$ $$logP(a_1,a_2,b) = \sum_{i=1}^{100} y_ip(cooktime_i, heat_i) + (1-y_i)(1-p(cooktime_i, heat_i))$$

由於P是data的likelihood,當likelihood越大表示使用的模型(由參數a,b,c代表)更能解釋觀察到的資料。因此理想上最好的隱藏因子，非常有可能在P最大的時候被呈現出來。這意味著瓦哥必須要求出一組參數$(a_1^*, a_2^*, b^*)$, 使的 $$logP(a_1^*,a_2^*,b^*) = max_{a_1,a_2,b} logP(a_1,a_2,b)$$

既然有東西要最佳化，那當然是請出gradient descent先。 於是瓦哥超嚴謹的計算了這個函數的gradient。 $$\frac{\partial }{\partial a} logP = \sum_i(y_i - P_i) x_i$$ $$\frac{\partial }{\partial b} logP = \sum_i(y_i - P_i) $$ 為了簡單起見，上面符號使用了

  1. $a = (a_1, a_2)$,
  2. $x = (cooktime_i, heat_i)$
  3. $P_i = p(cooktime_i, heat_i)$

下面的python code實現了這樣一個目標函數。

In [4]:
def sig_fun(x, w, b):
    ## input
    ## x: (d, n) np-array
    ## w: (n, ) np-array
    ## b: constant
    return 1.0 / (1.0 + np.exp(-x.dot(w) - b) )

class logistic_regression_fun:
    # function for -\sum_i y_i log(p) + (1-y_i) log(1-p)
    def __init__(self, x, y):
        self.y = y # (d, ) ndarray
        self.x = x # (d, n) ndarray
        
    def fval(self, weights):
        w, b = weights
        p = sig_fun(self.x, w, b)
        return -np.sum(self.y * np.log(p) - (1.0 - self.y) * np.log(1.0 - p))
    
    def grad(self, weights):
        w, b = weights
        delta = self.y - sig_fun(self.x, w, b)
        return -x.T.dot(delta), -np.sum(delta)

再請出gradient descent來尋找參數惹。

def gradient_descent(obj_fun, w, step_size, iters):
    for i in range(iters): 
        w = map(lambda x: x[0] - step_size * x[1], zip(w, obj_fun.grad(w))) # x + step_size * grad f(x)
    return w

## stack data in an array
x = np.vstack([cook_time, heat]).T

## objective function
obj_fun = logistic_regression_fun(x, y)

## initial weights
w0 = [np.array([0.0, 0.0]), 0.0]

## gradient descent
w_opt = gradient_descent(obj_fun, w0, 0.1, 10000)

a1, a2, b = (w_opt[0][0], w_opt[0][1], w_opt[1])
print "最佳參數 a1: %f, a2: %f, b: %f" % (a1, a2, b)

for i in range(2):
    plt.plot(cook_time[y==i], heat[y==i], '.')

x = np.linspace(-0.0, 1.5, num=1000)
plt.plot(x, -(a1/a2) * x - b/a2)
plt.xlabel('cook time')
plt.ylabel('heat')
plt.legend(['not good', 'good'])
plt.show()

最佳參數 a1: -40.576371, a2: 36.742738, b: 2.968381

上圖顯示了GD方法找到的參數以及根據這組參數建構出來的隱藏因子。結果和我們的直覺相符!
