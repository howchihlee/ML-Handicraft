異常資料問題¶

收到廢資料怎麼辦?

阿峰再次前往調查含氧量與植物生長密度的關聯。這次他蒐集到的資料長這樣：

In [1]:
import numpy as np
import matplotlib.pylab as plt
% matplotlib inline

o2_density = 0.2 + np.random.rand(40) * 0.8
plant_density = 3.0 * o2_density + 10.0 + 0.5 * np.random.randn(40)
o2_density[-1], plant_density[-1] = 0.22, 20.0

plt.plot(o2_density, plant_density, '.')
plt.xlabel('O2 density')
plt.ylabel('plant density')

plt.xlim((0, 1))
plt.ylim((8, 25))
plt.show()

一切似乎都很美好，數據依舊呈現線性關係，除了最後一次樣本採集出現了偏差。最後一次採集到的資料點遠遠偏離了整體資料的趨勢。無論如何，阿峰依舊大無畏的使用了線性迴歸來估計數據間線性關係。

讓我們稍微回味一下。所謂的線性關係指的是我們猜測植物密度和含氧量之間滿足了下面的數學式:

植物密度　＝　$a$ $\times$ 含氧量　＋　$b$

而我們的問題是，根據收集到的植物密度與含氧量數據，我們怎麼知道係數$a$與$b$呢？

一個可行的辦法是：尋找係數$ab$讓預測值與收集到的資料點誤差最小。換句話說，我們希望尋找係數$ab$，使得誤差函數 $$f(a, b) = \frac{1}{2}\sum_{i = 1}^{40} (ax_i + b - y_i)^2$$ 最小化。

下面的python code實做了上面的誤差函數。

In [2]:
class simple_linear_regression_fun:
    # function for 1/2 |a x_i + b - y_i|^2_2 
    def __init__(self, x, y):
        self.y = y # (n, ) ndarray
        self.x = x # (n, ) ndarray
        
    def fval(self, weights):
        a, b = weights
        return 0.5 * np.linalg.norm(a * self.x + b - self.y) ** 2
    
    def grad(self, weights):
        a, b = weights
        delta = (a * self.x + b - self.y) 
        return delta.dot(self.x), delta.sum()

為了讓誤差最小，阿峰也使用了梯度漸降法(Gradient descent, GD)來尋找最佳的係數$ab$

下面的python code實做了GD方法。

In [3]:
def gradient_descent(obj_fun, w, step_size, iters):
    for i in range(iters): 
        w = map(lambda x: x[0] - step_size * x[1], zip(w, obj_fun.grad(w))) # x + step_size * grad f(x)
    return w

為了讓程式簡潔起見，我們先把繪製結果的程序包裝起來。

In [4]:
def plot_result(obj_fun, w0, step_size):
    a10, b10 = gradient_descent(obj_fun, w0, step_size, 10)
    a20, b20 = gradient_descent(obj_fun, w0, step_size, 20)
    a1000, b1000 = gradient_descent(obj_fun, w0, step_size, 1000)
    xplot = np.linspace(0.0, 1.0, num = 10)

    plt.plot(o2_density, plant_density, '.')
    plt.plot(xplot, 3.0 * xplot + 10.0, 'r--', label = 'true slope')
    plt.plot(xplot, a10 * xplot + b10, 'k', label = '10 iterations')
    plt.plot(xplot, a20 * xplot + b20, 'g', label = '20 iterations')
    plt.plot(xplot, a1000 * xplot + b1000, 'y', label = '1000 iterations')
    plt.legend(bbox_to_anchor=(1.05, 1), loc=2, borderaxespad=0.)
    plt.xlabel('O2 density')
    plt.ylabel('plant density')
    plt.xlim((0, 1))
    plt.ylim((8, 23))
    plt.show()

萬事俱備，阿峰迫不期待的使用了誤差函數 + GD方法來分析他蒐集的數據。

In [5]:
obj_fun = simple_linear_regression_fun(o2_density, plant_density)
w0 = [0, 0]
step_size = 0.01

plot_result(obj_fun, w0, step_size)

出乎意料地，誤差函數 + GD 給出了一個糟糕答案！可以看到黃色的線是GD最後搜尋出的最佳解。但黃色的線卻和大部分資料點的理想趨勢(紅色虛線)不吻合！

阿峰猜測，這個不吻合的趨勢可能是因為最後一次測量帶來的巨大誤差,而誤導了GD。為了檢驗這個猜測，阿峰首先手動去除了最後一個資料點並再次使用了誤差函數 + GD來估計參數$ab$

In [6]:
obj_fun = simple_linear_regression_fun(o2_density[:-2], plant_density[:-2]) # 在這裡踢除了最後一個資料點

w0 = [0.0, 0.0]
step_size = 0.01

plot_result(obj_fun, w0, step_size)

這一次誤差函數 + GD順利地找到了理想中的參數！

想一想¶

我們可以每次都手動剔除不理想的數據嗎?

讀過一些文獻後，阿峰認為問題的根源來自於誤差函數中的平方。而對誤差取平方的結果，會放大不好預測的資料點在整個誤差函數的比重。舉個例來說，假設我們有五個資料點，他們的數值如下

x 1 1.5 2 2.5　 3
y 2.6 1.6 2.1 2.6 3.2

大家應該都看得出趨勢似乎是 x=y。要是使用這個線性模型的話，那麼我們得到各個資料點的誤差會是：

x 1 1.5 2 2.5　 3
y 2.6 1.6 2.1 2.6 3.2
y=x模型對y預測　 1.0 1.5 2.0 2.5 3.0
猜測y=x的平方誤差　 2.56 0.01 0.01 0.01 0.04

因此誤差的總和是　2.63。

那要是我們猜測 y全部等於2,那個個資料點的誤差則變成：

x 1 1.5 2 2.5　 3
y 2.6 1.6 2.1 2.6 3.2
y=2模型對y預測　 2.0 2.0 2.0 2.0 2.0
猜測y=2的平方誤差　 0.36 0.16 0.01 0.36 1.44

總體誤差成為了2.33，比前面合乎直覺的模型還好！ 從上面的例子我們可以看到，y=x的模型事實上後面四個資料點的預測都比y=2好，但因為第一個資料點的誤差是1.6，平方後使的整體的誤差超過2.5。所以，如果我們不要對誤差取平方，那少數幾個預測不好的點，也許就不會誤導整個誤差函數。

Huber 函數¶

Huber函數是這個想法下的產物。Huber函數大致上被刻劃成

  1. 當誤差在某個範圍內，我們對誤差取平方
  2. 當誤差太大時，我們取誤差的絕對值以降低異常值的比重

慎重起見，阿峰嚴謹地寫下了Huber函數的定義：

$h_\mu(x) = \frac{1}{2\mu}x^2$ if $|x|<\mu$

$h_\mu(x) = |x|-\mu/2$ if $|x|\geq\mu$

並超嚴謹地計算了Huber函數的梯度(gradient):

$\frac{d}{dx}h_\mu(x) = x/\mu$ if $|x|<\mu$

$\frac{d}{dx}h_\mu(x) = sgn(x)$ if $|x|\geq\mu$

然後超超超嚴謹地實做了Huber函數和梯度計算:

In [7]:
def huber(z, mu):
    zh = np.zeros_like(z)
    zh[abs(z) <= mu] = z[abs(z) <= mu] ** 2 / 2.0 / mu
    zh[abs(z) > mu] = abs(z[abs(z) > mu]) - mu / 2.0
    return zh

def huber_grad(z, mu):
    zh = np.zeros_like(z)
    zh[z > mu] = 1.0
    zh[z < -mu] = -1.0
    zh[abs(z) < mu] = z[abs(z) < mu]/ mu
    return zh

讓我們來看看這個新的誤差函數長什麼樣子。

In [8]:
import matplotlib.pyplot as plt
import numpy as np
%matplotlib inline

x = np.linspace(-2.0, 2.0, num=100)
plt.plot(x, huber(x, 1.0),'r', label = 'mu = 1')
plt.plot(x, huber(x, 0.5),'g', label = 'mu = 0.5')
plt.plot(x, huber(x, 0.1),'b', label = 'mu = 0.1')
plt.legend(bbox_to_anchor=(1.05, 1), loc=2, borderaxespad=0.)
plt.show()

上面不同顏色的線畫出了huber函數如何隨著參數$\mu$變化。可以看到，

  1. huber函數的值大致上是和離原點的距離成比例。遠離原點兩倍遠，huber函數的值也大致上增加為兩倍。
  2. 無論參數$\mu$是多少, huber函數都在原點達到最小值0。
  3. 在靠近原點的時候，huber函數從直線逐漸轉變為光滑的弧線。$\mu$越大，轉變發生的範圍越大。

讓我們再來看一下上面的例子。但這一次我們使用Huber函數來選取最佳的參數。

回味一下我們假想中的資料:

x 1 1.5 2 2.5　 3
y 2.6 1.6 2.1 2.6 3.2

假設我們使用Huber函數並設定參數$\mu=1$。也就是說，在誤差大於1時我們希望降低這個資料點的比重。我們可以計算y=x這個模型的誤差：

x 1 1.5 2 2.5　 3
y 2.6 1.6 2.1 2.6 3.2
y=x模型對y預測　 1.0 1.5 2.0 2.5 3.0
猜測y=x的Huber誤差 0.18 0.08 0.005 0.18 0.7

因此誤差的總和是　1.135。

那要是我們猜測 y全部等於2，那這些資料點的誤差則變成：

x 1 1.5 2 2.5　 3
y 2.6 1.6 2.1 2.6 3.2
y=2模型對y預測　 2.0 2.0 2.0 2.0 2.0
猜測y=2的Huber誤差　 0.18 0.08 0.005 0.18 0.7

總體誤差成為了1.145。

這一次，合乎直覺的模型預測誤差比較小！

新的目標函數¶

有了Huber函數，阿峰重新設計了新的目標函數： $$f(a, b) = \sum_{i = 1}^{40} h_\mu(ax_i + b - y_i)$$ 這個目標函數類似於之前的線性迴歸。對於每一個資料$x_i$ 我們首先計算預測誤差$ax_i + b - y_i$，並用Huber函數來衡量這個誤差的比重。而我們希望找尋參數$ab$，使得整體誤差的和最小化。

下列的python code實現了這個新的目標函數。

In [9]:
class simple_huber_regression:
    def __init__(self, x, y, mu):
        self.x = x
        self.y = y
        self.mu = mu
        return 
    
    def fval(self, weights):
        (a, b) = weights
        return np.sum(huber(a * self.x + b - self.y, self.mu))
    
    def grad(self, weights):
        (a, b) = weights
        delta = huber_grad(a * self.x + b - self.y , self.mu)
        return  delta.dot(self.x), delta.sum()

阿峰於是乎使用GD+新的目標函數來尋找最佳參數。這一次，阿峰並沒有剔除任何數據。

In [10]:
obj_fun = simple_huber_regression(o2_density, plant_density, 1.0)

w0 = [0.0, 0.0]
step_size = 0.01

plot_result(obj_fun, w0, step_size)

有了Huber函數的幫忙，GD順利地在不完美的數據中找到理想中的參數。阿峰滿意惹。
