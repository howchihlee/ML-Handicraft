爬山問題：¶

如何快速爬到最高峰？

阿峰正在某森林遊樂區享受新鮮空氣。突然間的肚痛如絞讓阿峰想立即回到山頂的遊客中心尋求解脫。阿峰該如何快速返回山頂？

一個很直覺想法是，如果沿著最陡峭的坡邁進，應該可以快速的到達山頂吧？Gradient descent (GD, 梯度下降)就是建立在這個直覺想法上的演算法。我們今天就跟著嚮導GD，來爬數學的山。至於阿峰，人生處處有解脫。

先來簡單定義一下數學的山。數學的山可以被描述成一個函數，這個函數f描述了每一個座標點x對應的高度f(x)。 而一個函數的梯度(gradient)則指出了最陡峭的坡的走向。

梯度下降(GD)：

  1. 從某地點開始
  2. 計算梯度
  3. 沿著梯度方向邁進一小步
  4. 重複2

GD就就像是在爬山一樣。GD在移動前會先尋找坡度最陡的方向(第二步)，然後沿著那個方向邁出一小步(第三步)。移到了新的地點，GD會再次尋找坡度最陡的方向(第二步)並往下一個地點(第三步)邁進，直到爬到適合的高度。

大家也許會有疑惑，例如：

  1. 從哪裡開始走？我家住比較遠有差嗎？
  2. 該走多大步？長得高步伐大有差嗎？
  3. 該走幾步？可以不走躺著上山嗎？

讓我們先實做一個GD演算法，再透過做實驗來驗證這些問題。

利用python可以簡單三行來實現GD algorithm。

In [1]:
def gradient_descent(obj_fun, w, step_size, iters):
    for i in range(iters): 
        w = map(lambda x: x[0] - step_size * x[1], zip(w, obj_fun.grad(w))) # x + step_size * grad f(x)
    return w

gradient_descent這個function有四個arguments:

1. obj_fun: 一個object 描述了"數學的山"。
2. w0: 初始值，亦即旅程的出發點。
3. step_size: 沿著梯度每一小步的大小
4. iters: 總共要移動的步數。

後面會接著討論這些arguments如何影響GD的行為。

看到這裡大家也許會想，這一切到底跟機器學習哪裡相關？這個連結就在於如何設計obj_fun。接下來我們會討論如何設計obj_fun來完成第一個ML algorithm:　線性迴歸(linear regression)

簡單線性迴歸¶

阿峰在森林遊樂區其實是有目的的：他想研究空氣含氧量與植物密度的關係。我們來看看他蒐集的資料。

阿峰非常幸運，他收集到的數據呈現了強烈的相關性。他猜測，其實含氧量跟植物密度間平均上呈現了線性的關係，也就是說阿峰認為平均上而言

植物密度　＝　a ｘ 含氧量　＋　b

因為技術上的困難，阿峰並無法測量到含氧量低於0.2的數據。如果能知道係數$a$與$b$，阿峰便能預測當含氧量是0的時候，植物的平均生長密度是　$a\times 0＋b=b$。問題是，怎麼知道係數$a$與$b$呢？

誤差函數¶

阿峰查找文獻後採用了Legendre的想法：尋找係數$ab$讓預測值與收集到的資料點誤差最小。阿峰定義了這樣一個函數： $$f(a, b) = \frac{1}{2}\sum_{i = 1}^{100} (ax_i + b - y_i)^2$$

其中：

  1. i從1到100代表了100個測量到的資料點
  2. $x_i$是其中某一次測量到的含氧量
  3. $y_i$是相對應的植物生長密度

在這個式子中，$(ax_i + b - y_i)^2$ 代表了預測值$ax_i + b$ 與測量數據$y_i$的差距。當這個值越大，表示使用參數$ab$所作的預測越不準確。因此，透過把最小化上面的誤差函數降，我們就可以選取一組最準確的參數$ab$。

經過一些計算，這個函數的梯度可以表示成

$$\nabla f (a, b) = (\sum_{i = 1}^{100} (ax_i + b - y_i) x_i, \sum_{i = 1}^{100} (ax_i + b - y_i))$$

阿峰發現了第一座ＭＬ的山。

下面的python code 實現了我們想要的函數。

In [3]:
class simple_linear_regression_fun:
    # function for 1/2 |a x_i + b - y_i|^2_2 
    def __init__(self, x, y):
        self.y = y # (n, ) ndarray
        self.x = x # (n, ) ndarray
        
    def fval(self, weights):
        a, b = weights
        return 0.5 * np.linalg.norm(a * self.x + b - self.y) ** 2
    
    def grad(self, weights):
        a, b = weights
        delta = (a * self.x + b - self.y) 
        return delta.dot(self.x), delta.sum()

走幾步有差嗎？¶

定義好了目標函數，讓我們實驗一下GD嚮導會不會迷路。我們其實知道阿峰蒐集的資料隱藏的真實係數是a=3，b=10。先來看看當我們跟著GD嚮導走10，20，100步，會走到哪去。

In [4]:
obj_fun = simple_linear_regression_fun(o2_density, plant_density)

w0 = [0.0, 0.0]
step_size = 0.01

a10, b10 = gradient_descent(obj_fun, w0, step_size, 10)
a100, b100 = gradient_descent(obj_fun, w0, step_size, 20)

誤差真的有下降嗎？¶

從上面的圖可以看到，GD確實找出了隱藏的係數ab。上圖中紅色的虛線代表了我們預設的，含氧量與植物密度的線性關係。黑線，綠線，黃線則代表了GD走10, 100, 1000步後解出的線性關係。這表示GD的確逐漸靠近了預設的係數。我們可以更進一步來看看GD是否降低了預測值與資料點間的誤差。

動手作:¶

我們已經實驗了步數如何影響GD的行為，接下來兩個問題請大家來實驗看看

  1. 輸入的起始點w0會影響GD找到解答嗎?
  2. 輸入的參數step_size會影響GD找到解答嗎?

事實上GD並不是完美的。動手之後應該可以發現：

  1. w0在哪裡其實並無差異,只要走的步數夠多終究能抵達終點。
  2. 但是 step_size 太大, GD是會迷路的。

關於這些比較理論的分析，我們留待以後再討論。
